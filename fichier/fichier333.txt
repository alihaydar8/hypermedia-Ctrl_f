Conférences industriellesCompte rendu de la conférence présenter par monsieur Khaled BEGREDJ, ancien élevé à paris8, sur son parcours et les outils utiliser dans une carrière d’un développement, du 20 octobre 2021 à l’université paris8.Le conférencier a eu un master informatique mention machine embarquée à l’université paris8 en 2008. La formation de machine embarquée lui a préparé à beaucoup de choix mais ces deux stages pendant son master chez orange ont orienté sa carrière vers le développement.Khaled a commencé par la présentation de son parcours des postes commencent par XL AIRWAYS finissons par Air France passant par CSTB. Khaled a débuté comme Ingénieur d’études & développement puis il a grimpé l’échelle pour devenir un chef de projet comme tout développeur qui réussit dans son domaine.Monsieur BEGREDJ a présenté tout long du conférence toute une série des outils et techniques informatiques utilisées par les développeurs au niveau de 7 états. Code (source code mangement), Build (continuous integration ), Test (continuous testing ), Package (artifact repository), Release (release automation), Configure (infrastructure as code), Monitor (applications performance monitoring). Les technologies utilisent :La plateforme de travail collaborative est un espace de travail virtuel. C'est un outil qui centralise tous les outils liés à la conduite d'un projet, la gestion des connaissances ou au fonctionnement d'une organisation et les met à disposition des acteurs. On note Slack, Teams, Workplace.Le système de suivi des bugs est un logiciel qui permet d'effectuer un suivi des bugs signalés dans le cadre d'un projet de développement de logiciel. Il permet d'aider les utilisateurs et les développeurs à améliorer la qualité d'un logiciel. Les plus fameux bugs tracker sont Jira, Sharepoint, Alfresco.Les hébergeurs du code et les logiciels du suivie du code comme git sont maintenant une prioritaire dans tous projets avec plus que deux développeurs pour suivre le code et savoir le propriétaire du chaque bout de code et en cas de problème pour pouvoir retourner en arrière et savoir d’où provient le problème et on site Github, Gitlab, Bitbucket comme meilleur hébergeur du code.Gitflow est un modèle de branching Git alternatif qui utilise des branches de fonctionnalité et plusieurs branches primaires, est parfaitement adapté aux projets avec un cycle de livraison planifié. Les développeurs créent une branche de fonctionnalité et attendent que la fonctionnalité soit terminée pour la merger à la branche principale. Ces branches de fonctionnalités au long cours nécessitent une plus grande collaboration lors du merge, car elles risquent des conflits. Ce modèle est normalement utilisé avec le développement agile par exemple avec le Framework Scrum qui créer des sprints « cycle » qui sera plus ou moins la dure de vie de la branche créer lors du Gitflow.Maintenant c’est le tour de l’intégration continue, un ensemble de pratiques utilisées en génie logiciel consistant à vérifier qu’après chaque modification de code source le code compile toujours. Les débutants ne voient pas l’importance de l’intégration continue mais c’est un important pour livre un produit fonctionnelle. Il est assuré par plusieurs outil par exemple jenkins, Bamboo, Apache, Github Action, Gitlab CI.L’inspection continue comme son l’indique c’est l’inspection pour vérifier que toute est en ordre commençons par les noms des classes en passant par les fail de la sécurité jusqu’au l’optimisation du code.Pour conclure, pour entrer dans le monde de l’informatique est devenir un bon développeur on doit connaitre quelque technique, technologie de la base du développement par exemple la conception du repository, du commit, du push, pull et la migration des branches avec leurs conflits.Fait le 25 /10/2021 à ParisAli HAIDAR-AHMADM1-Informatique
